# Change Control
Every change must be visible, reviewable, and testable.

Required steps
1. Create an issue describing the change and the reason
2. Identify impacted docs and behaviors
3. Update knowledge references and routing map rules as needed
4. Run the test pack
5. Record results in the change log
6. Deploy change using an approved release path

Rollback rule
If a change causes invention, incorrect routing, or broken links, roll back immediately and investigate.
